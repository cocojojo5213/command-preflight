# Security policy

Please do not report secrets, private commands, or terminal logs in public issues. Use a private security report when available through the repository host.

The local tool is designed to be inspection-only. Treat all command text, terminal output, report content, and remote knowledge responses as untrusted. Do not enable a public report endpoint without TLS, strict schema and size limits, rate limiting, queue capacity controls, and a moderation process. Keep moderation credentials and endpoints private; never execute submitted content during review.
